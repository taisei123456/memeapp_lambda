# Step 8: 動作確認（受け入れ基準チェック）

この手順では、要件どおりに全体がつながっているかを確認します。

## 目的
1. 各サービスが正しく連携しているかを見る。
2. 失敗時の挙動も確認する。

## 確認する順番
1. フロントURLにアクセスして UI が出る。
2. 画像アップロードで Input_Bucket に保存される。
3. Processor_Lambda が自動起動する。
4. Output_Bucket に合成画像が保存される。
5. フロントで合成画像が表示される。
6. 失敗ケースで UI と CloudWatch にエラーが出る。

## 手順 1: フロント画面を確認する
1. Frontend_Bucket の Website endpoint を開きます。
2. UI が表示されるか確認します。
3. レイアウトが崩れていないか見ます。

## 手順 2: アップロードを確認する
1. 画像を選択します。
2. アップロードを実行します。
3. Input_Bucket に元画像が入るか確認します。

## 手順 3: Lambda の起動を確認する
1. CloudWatch Logs を開きます。
2. Processor_Lambda のログを確認します。
3. ObjectCreated で起動しているか見ます。

## 手順 4: 合成結果を確認する
1. Output_Bucket を開きます。
2. memes/ 配下に画像があるか確認します。
3. フロント側で画像が表示されるか確認します。

## 手順 5: エラー時の確認
1. API Gateway が失敗していないか確認します。
2. Lambda が例外を出していないか確認します。
3. CORS エラーが出ていないかブラウザで確認します。

## 確認ポイント
1. 要件どおりの順番で動く。
2. 失敗時に原因が追える。
