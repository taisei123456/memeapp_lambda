# Step 7: フロントエンド実装（最小）

この手順では、アップロードから表示までを動かす最小構成のフロントエンドを用意します。

## 目的
1. 画像を選ぶ。
2. API Gateway から Presigned URL を取得する。
3. Input_Bucket に画像をアップロードする。
4. Output_Bucket の完成画像を表示する。

## 画面要素
1. ファイル選択
2. アップロード開始ボタン
3. ステータスメッセージ
4. 結果画像表示領域

## 処理フロー
1. `GET /get-upload-url` を呼びます。
2. 返ってきた `uploadUrl` に PUT で画像を送ります。
3. 成功後、`resultUrl` を使って画像の取得を試します。
4. まだ生成されていなければ数秒おきに待ちます。
5. 取得できたら画像を表示します。

## 実装上のポイント
1. 失敗時は必ず UI にメッセージを出します。
2. 画像がない場合は送信しません。
3. ポーリングはやりすぎないように回数を決めます。

## 配置するファイル
1. `index.html`
2. `app.js`
3. `style.css`

## 置き方
1. 3ファイルを Frontend_Bucket にアップロードします。
2. `index.html` から `style.css` と `app.js` を参照します。
3. Website endpoint で表示確認をします。

## 確認ポイント
1. 画面が表示される。
2. 画像を選べる。
3. API の結果を使ってアップロードできる。
4. 結果画像を表示できる。
