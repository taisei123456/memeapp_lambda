# Step 9: コストと運用の最低限対策

この手順では、Learner Lab の予算上限と運用負荷を抑えるための設定を整理します。

## 目的
1. 余計な課金を避ける。
2. 使わないリソースを溜めない。

## 対策 1: Lambda timeout を短めにする
1. Lambda の timeout は 10〜30 秒から始めます。
2. 画像サイズが大きい場合だけ長めに調整します。

## 対策 2: Lambda memory を適切にする
1. Processor_Lambda は 512MB 前後から始めます。
2. 足りなければ上げます。
3. 必要以上に大きくしすぎないようにします。

## 対策 3: S3 ライフサイクルで古い画像を消す
1. Input_Bucket と Output_Bucket にライフサイクルルールを入れます。
2. 例として 7 日で削除します。
3. テスト画像が溜まり続けるのを防ぎます。

## 対策 4: CloudWatch Logs の保持期間を短くする
1. ロググループの保持期間を 7 日など短めにします。
2. 不要なログで容量を使い続けないようにします。

## 対策 5: API Gateway の不要なステージを消す
1. 使わないステージは削除します。
2. 古いデプロイを残しっぱなしにしないようにします。

## 運用の基本
1. 使い終わったら確認用の画像を削除します。
2. テスト時の設定を本番用に戻します。
3. 何がどこに保存されるかを固定しておきます。

## 確認ポイント
1. 課金対象が増えすぎていない。
2. ログと画像が溜まり続けていない。
3. 不要な API ステージが残っていない。
